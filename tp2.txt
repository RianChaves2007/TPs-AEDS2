Modelagem de Dados: Restaurante

Neste trabalho, voce deverá realizar a modelagem de dados de um sistema que representa uma coleção
de restaurantes distribuı́dos em diferentes cidades do mundo.
   O objetivo e definir estruturas de dados que permitam representar adequadamente as informações
de cada restaurante, servindo como base para etapas futuras do trabalho, nas quais serao aplicados
algoritmos e estruturas de dados como listas, filas, árvores, além de operacões de pesquisa e ordenação.
Objetivo

Projetar e implementar os seguintes tipos de dados:

   • Restaurante

   • Data

   • Hora

   Esses tipos devem ser definidos de forma clara e organizada, permitindo representar fielmente os
dados provenientes de um arquivo CSV contendo informações sobre restaurantes.

Estilo de codificação

Para a codificação, deve-se utilizar as seguintes convenções de nomenclatura:

   • Java: camel case

   • C: snake case

   Tipos (class e struct) devem iniciar com letra maiúscula, enquanto variáveis e métodos (funções)
devem iniciar com letra minúscula.

Especificação dos Tipos
Tipo Data

Representa uma data no calendário.

   • ano (inteiro)

   • mes (inteiro)

   • dia (inteiro)

Tipo Hora

Representa um horário do dia.

   • hora (inteiro, de 0 a 23)

   • minuto (inteiro, de 0 a 59)

Tipo Restaurante

Representa um restaurante com suas caracterı́sticas principais.

   • id: identificador unico (inteiro)

   • nome: nome do restaurante (string)
   • cidade: cidade onde esta localizado (string)

   • capacidade: capacidade maxima de clientes (inteiro)

   • avaliacao: nota media (double)

   • tipos cozinha: colecao de tipos de culinaria (arranjo/lista de strings)

   • faixa preco: nivel de preco (inteiro de 1 a 4), onde:

         – 1→$

         – 2 → $$

         – 3 → $$$

         – 4 → $$$$

   • horario abertura: horario de abertura (tipo Hora)

   • horario fechamento: horario de fechamento (tipo Hora)

   • data abertura: data de abertura (tipo Data)

   • aberto: indica se o restaurante esta em funcionamento (booleano)

Regras de Mapeamento (CSV → Estruturas)

Ao ler o arquivo CSV:

   • O campo tipos cozinha deve ser convertido em um arranjo de strings, utilizando o separador
     ;

   • O campo faixa preco deve ser convertido para um inteiro de 1 a 4

   • O campo horario (formato HH:mm-HH:mm) deve ser dividido em:

         – horario abertura

         – horario fechamento

   • O campo data abertura (formato YYYY-MM-DD) deve ser convertido para o tipo Data

   • O campo aberto deve ser interpretado como valor booleano

 11.        Lista com Alocação Sequencial em Java: Crie uma Lista de registros baseada na de
       inteiros vista na sala de aula. Sua lista deve conter todos os atributos e métodos existentes na
       lista de inteiros, contudo, adaptados para a classe Restaurante. Lembre-se que, na verdade,
       temos uma lista de ponteiros (ou referências) e cada um deles aponta para um registo. Neste
       exercı́cio, faremos inserções, remoções e mostraremos os elementos de nossa lista.

       Os métodos de inserir e remover devem operar conforme descrito a seguir, respeitando parâmetros
       e retornos. Primeiro, o void inserirInicio(Restaurante restaurante) insere um registro
       na primeira posição da Lista e remaneja os demais. Segundo, o void inserir(Restaurante
       restaurante, int posicao) insere um registro na posição posicao da Lista, onde posicao < n e
       n é o número de registros cadastrados. Em seguida, esse método remaneja os demais registros. O
       void inserirFim(Restaurante restaurante) insere um registro na última posição da Lista.
       O Restaurante removerInicio() remove e retorna o primeiro registro cadastrado na Lista
       e remaneja os demais. O Restaurante remover(int posicao) remove e retorna o registro
       cadastrado na posição informada da Lista e remaneja os demais. O Restaurante removerFim()
       remove e retorna o último registro cadastrado na lista.

       A entrada padrão é composta por duas partes. A primeira parte é igual a entrada da primeira
       questão e os registros presentes devem ser inseridos ao final da Lista. As demais linhas corres-
       pondem a segunda parte. A primeira linha da segunda parte é composta de um número inteiro n
       indicando a quantidade de registros a serem inseridos/removidos. Nas próximas n linhas, tem-se
       n comandos de inserção/remoção a serem processados neste exercı́cio. Cada uma dessas linhas
      tem uma palavra de comando: II inserir no inı́cio, I* inserir na posição informada, IF inserir
      no fim, RI remover no inı́cio, R* remover na posição informada e RF remover no fim. No caso
      dos comandos de inserção, temos também o identificador do registro a ser inserido. No caso
      dos comandos de inserção na posição informada, a posição fica imediatamente após a palavra de
      comando. A saı́da padrão tem uma linha para cada registro removido, sendo que essa informação
      será constituı́da pela palavra “(R)” e o atributo nome do registro removido. No final, a saı́da
      mostra os registros presentes na Lista do primeitro ao último.
